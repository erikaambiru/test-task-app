# タスク管理アプリケーション開発のためのプロンプト集

## 1. プロジェクト初期設定プロンプト

```

【フロントエンド】
- React 18とTypeScriptを使用
- Chakra UIでのモダンなUI実装
- Framer Motionでのアニメーション
- @hello-pangea/dndでのドラッグ＆ドロップ機能
- Axiosでのバックエンド通信

【バックエンド】
- Go言語でRESTful API
- Gorilla Muxでのルーティング
- ローカルJSONファイルでのデータ保存
- タスクのCRUD操作（作成、読み取り、更新、削除）

プロジェクトの初期設定に必要なコマンドと、基本的なディレクトリ構造を教えてください。
```

## 2. タスクモデル設計プロンプト

```
タスク管理アプリのタスクモデルを設計したいです。以下の要件を満たすTypeScriptの型定義とGoの構造体を教えてください：

【タスク属性】
- ID（一意の識別子）
- タイトル
- 説明（オプション）
- ステータス（未着手、進行中、完了）
- 優先度（高、中、低）
- 作成日時
- 更新日時

【必要な型定義】
- TaskStatus型（todo, in-progress, done）
- TaskPriority型（high, medium, low）
- TaskSortType型（優先度順、作成日順、更新日順、タイトル順）
- Task型（上記のすべての属性を含む）

フロントエンド（TypeScript）とバックエンド（Go）の両方で使用できる型定義と構造体を教えてください。
```

## 4. バックエンド開発プロンプト

```
Go言語を使用してタスク管理アプリのバックエンドAPIを開発しています。以下の機能を実装するためのコードと構造を教えてください：

1. メインサーバープログラム（main.go）
   - HTTPサーバーの設定
   - Gorilla Muxによるルーティング設定
   - CORSの設定

2. APIエンドポイント
   - GET /tasks - すべてのタスクを取得
   - GET /tasks/:id - 特定のタスクを取得
   - POST /tasks - 新しいタスクを作成
   - PUT /tasks/:id - タスクを更新
   - DELETE /tasks/:id - タスクを削除

3. データモデルとストレージ
   - タスクの構造体定義
   - JSONファイルへの保存と読み込み
   - IDの生成方法

RESTful APIの原則に従い、エラーハンドリングも含めた実装方法を教えてください。
```
